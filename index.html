Project framing

Speed Feedback Message Signs are being examined across Maine as a practical speed-management and safety-support tool. This page is designed for GitHub deployment and can function as a project website, portfolio entry, or field-visit dashboard.

Suggested sections for your final research site

  • Map of sign locations, grouped by field-visit route or district.
  • Site inventory table with coordinates and visit order.
  • Methodology, data collection, before-after speed analysis, and findings.
  • Photos, installation status, and future policy or engineering recommendations.

How the current grouping works

The current KML uses repeated visit numbering, so this page automatically creates a new trip group each time the numbering resets. That makes the dashboard flexible: if you later replace the KML with a fuller statewide file, the page will still adapt.

GitHub Pages notes

This page is designed as a single static HTML file plus your KML file. No build step is required. Upload both files to the same repository root and enable GitHub Pages.

  1. Create a GitHub repository.
  2. Upload index.html and DSFS_SiteVisitMap.kml.
  3. Go to Settings → Pages.
  4. Select the branch and root folder.
  5. Publish the site.
Easy future edits
  • Change the statewide total in the PROJECT.totalProjectSites constant.
  • Replace the KML with a fuller statewide file when all 92 locations are available.
  • Add a photos folder and link images to each site later.
  • Insert your results section after you complete the speed analysis.
